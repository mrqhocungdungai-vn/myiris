# Bundled skill/command snapshots — attribution and refresh

The directories under `claude-skills/` and `claude-commands/` are vendored
snapshots of third-party Claude Code skills, bundled so the pipeline's
"Install missing" action (SetupPanel) can provision them without network
access. They are **not** generated by this project; do not edit their content
directly — refresh the snapshot instead (see below).

## Why bundled instead of fetched live

Iris's "Install missing" action must work offline and without shelling out to
`npx`/`git` from inside a packaged Electron app (no interactive terminal to
fall back on). Vendoring a snapshot trades "always latest" for "always
works" — see the `pipeline-setup-install` capability spec for the
scenarios this satisfies.

**Trade-off, accepted explicitly:** these snapshots can lag the upstream
projects' latest releases. Skills are prompt content, not executable code —
staleness degrades guidance quality, not safety. Users who want the newest
upstream versions can always install them directly (via `skills.sh` /
`openspec init`); the installer never overwrites an existing entry, so a
manual newer install always wins.

## Refresh procedure

Run this on a dev machine with both tools available, then re-copy into this
repo and update the version/commit table above:

```bash
# mattpocock skills (grilling, tdd, code-review, diagnosing-bugs)
npx skills@latest add mattpocock/skills
# then copy the resolved skill directories from ~/.claude/skills/<name>
# (resolve symlinks — skills.sh installs them as symlinks into ~/.agents/skills/)

# OpenSpec skills + /opsx commands
mkdir -p /tmp/opsx-snapshot && cd /tmp/opsx-snapshot
openspec init /tmp/opsx-snapshot --tools claude
# then copy .claude/skills/openspec-* and .claude/commands/opsx/*.md
```

Copy the refreshed directories over `claude-skills/<name>/` and
`claude-commands/opsx/*.md` in this repo, update the commit/version column
above, and note the refresh date in the next commit message.

```bash
# LLM-Wiki skills (wiki-config, wiki-ingest, wiki-query, wiki-lint, wiki-integrate, wiki-crystallize)
# Prefer a tagged release over a floating branch commit — check
# https://github.com/vanillaflava/llm-wiki-skills/releases for the latest tag:
git clone --branch <tag, e.g. v0.5.0> --depth 1 https://github.com/vanillaflava/llm-wiki-skills.git /tmp/llm-wiki-skills
# then copy each of the 6 skill directories (each contains its own SKILL.md)
```

Copy the refreshed `wiki-*/` directories over `claude-skills/wiki-*/` in this
repo (verbatim, no content edits — the vault path is pinned externally via
`--append-system-prompt`, never by editing these snapshots), update the
release tag/commit column above, and note the refresh date in the next commit
message.
