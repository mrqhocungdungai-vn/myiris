## Purpose

Bundles the third-party skills and commands the PO/DEV personas invoke as repo-vendored snapshots, and provides a single explicit install action that provisions a fresh machine's `~/.claude` in one click — personas sync-installed (Iris-owned), everything else copied only where missing, so no existing tool-managed install (skills.sh, manual `openspec init`) is ever overwritten.

## Requirements

### Requirement: Required third-party skills ship as bundled snapshots

The repo SHALL bundle, under `resources/skills/`, snapshots of the third-party capabilities the pipeline personas invoke: the mattpocock skills (`grilling`, `tdd`, `code-review`, `diagnosing-bugs`), the OpenSpec workflow skills (all 6 generated by `openspec init --tools claude`), and the 6 `/opsx` command files. An `ATTRIBUTION.md` beside them SHALL record each snapshot's upstream source, version/commit, MIT license notice, and the manual refresh procedure. Bundled snapshots MAY lag upstream releases; refreshing them is an explicit maintainer task, never an automatic runtime fetch.

#### Scenario: Snapshots are self-contained

- **WHEN** the installer runs on a machine with no network access
- **THEN** all skills and commands install successfully from the bundled snapshots alone

#### Scenario: Attribution is present

- **WHEN** a contributor inspects `resources/skills/`
- **THEN** they find the upstream source, pinned version/commit, license notice, and refresh steps for every bundled set

### Requirement: One-click installer with two write policies

The app SHALL provide a single explicit install action (`installPipelinePrereqs`) that, in one run: installs the Iris personas via the existing agents install (sync policy — Iris-owned files are overwritten to match the bundle), and copies each bundled skill directory and `/opsx` command file into `~/.claude/skills` and `~/.claude/commands/opsx` **only where the destination does not already exist** — an existing entry, including a symlink, SHALL be skipped untouched. The action SHALL return a structured report (installed/skipped/errors per category) and SHALL run only on explicit user action, never at app startup.

#### Scenario: Fresh machine gets everything in one click

- **WHEN** a user with the Claude and openspec CLIs installed but an empty `~/.claude` clicks the install action
- **THEN** the personas, the 10 bundled skills, and the 6 `/opsx` commands are all installed, and the report lists them as installed

#### Scenario: Tool-managed installs are never clobbered

- **WHEN** `~/.claude/skills/tdd` already exists as a skills.sh-managed symlink and the install action runs
- **THEN** that entry is skipped and reported as already present, and only the missing entries are copied from the snapshots

#### Scenario: Personas still sync

- **WHEN** the install action runs on a machine with an outdated `~/.claude/agents/iris-po.md`
- **THEN** the persona is overwritten to the bundled version (existing agents-install behavior), while third-party skills keep their install-only-missing policy

#### Scenario: No writes without user action

- **WHEN** the app starts up on any machine
- **THEN** nothing is written to `~/.claude`; the installer runs only from the explicit UI action

### Requirement: Setup and workflow guide is documented bilingually

The repo SHALL contain a voice-first pipeline guide in English (`docs/PIPELINE_GUIDE.md`) and Vietnamese (`docs/PIPELINE_GUIDE.vi.md`), cross-linked, sharing one section structure: what the pipeline is, setup steps ending at the in-app install action, a walkthrough of what to say at each stage (PO grilling → "propose" → DEV "implement remaining tasks" → archive) that explains `/opsx:propose`/`/opsx:apply` are run by the agents rather than typed by the user, an appendix on using the agents directly in Claude Code, and troubleshooting mapped to the SetupPanel check rows. The README pipeline section SHALL link to the guide instead of duplicating it.

#### Scenario: A new user reaches a working pipeline from the guide alone

- **WHEN** a community user follows the guide's setup section on a machine with none of the prerequisites
- **THEN** every step is covered in order (Claude CLI, `claude setup-token`, openspec CLI, in-app install) and ends with the SetupPanel checks green

#### Scenario: Voice walkthrough teaches speaking, not typing

- **WHEN** the user reads the walkthrough section
- **THEN** each pipeline stage is described as what to say to Iris and what to expect back, and the `/opsx` commands are explained as the agents' internal mechanism
