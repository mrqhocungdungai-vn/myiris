# glass-hud-overlay

## Why

The three upstream ports (Deep Space restructure, two-hand gestures + orb, voice UI + setup) are implemented and archived — the deliberately deferred piece is upstream's Glass HUD: a second UI mode where Iris floats over the whole desktop as a click-through glass overlay, so the user keeps working in their apps while Iris stays ambient (orb, tasks column, comms, camera dock), interactive only where the pointer is over a `.hud-hit` element. All renderer prerequisites now exist in our tree (HudShell's imports — ReactorCore with micro-expressions, WorkCard + StepTimeline, HandSkeleton, tasks lib, wake word, sounds, lucide-react). What's missing is the mode itself plus its main-process support (overlay window behavior, tray, global hotkey).

## What Changes

- **HUD mode in the renderer**: adopt upstream `HudShell.tsx` (~270 lines) and `src/styles/hud.css` (~468 lines, the stylesheet intentionally skipped during `ui-deepspace-restructure`); App gains `uiMode` (`deck` | `hud`) + mode transition state and renders HudShell or the deck accordingly.
- **Click-through orchestration**: everything in HUD mode is pointer-transparent except `.hud-hit` elements; the renderer reports hover-over-interactive via `hud:interactive`, and main toggles `mainWindow.setIgnoreMouseEvents(!on, { forward: true })`.
- **Mode switching, three ways**: IPC `hud:toggle` (invoked from UI buttons), a global hotkey (`IRIS_HUD_HOTKEY`, default `Alt+Space`), and a macOS tray (menu-bar) presence with toggle/wake/sleep items (`build/trayTemplate.png` + `@2x`, generated by upstream's `scripts/render-icon.mjs`). Main emits `hud:mode` so the renderer animates deck↔HUD transitions; tray menu items reuse the existing `iris:sleep` path and add `iris:wake`.
- **Claude adaptations (Hermes → Claude)**: HUD task cards keep our agent (PO/DEV) badge, model, ⛓ chain badge, and step timeline; HUD "Tasks" column reflects our Work Stream data unchanged. **The PO question banner renders in HUD mode as a `.hud-hit` surface** so the voice-decision-relay click path keeps working while floating. Pipeline/role switching, model popover, SessionSwitcher, ProjectBar, and SetupPanel stay deck-only — the HUD offers an exit-to-deck control for management actions (matching upstream's minimal-HUD philosophy).
- **Window/lifecycle plumbing**: overlay-capable BrowserWindow flags, fullscreen-overlay swap on toggle, `win:control` channel for the windowed controls parity with upstream TopBar (if not already present), `globalShortcut` registration with failure logged as a sidecar event, unregister on quit.

## Capabilities

### New Capabilities
- `glass-hud-mode`: The HUD overlay mode itself — entering/leaving, click-through behavior with `.hud-hit` islands, layout (orb cluster, collapsible tasks column, comms, camera dock), and parity of Claude-specific surfaces (task cards, PO question banner) inside the HUD.
- `hud-activation`: The three activation surfaces — renderer toggle via `hud:toggle`, global hotkey `IRIS_HUD_HOTKEY`, and the tray presence with HUD/wake/sleep controls.

### Modified Capabilities

- `voice-decision-relay`: add the requirement that a pending PO question remains answerable (voice AND click/dwell) while in HUD mode — the banner is a `.hud-hit` element in the overlay.

## Impact

- **Renderer**: new `src/components/HudShell.tsx`, new `src/styles/hud.css` (+ import in `styles/index.css`); `App.tsx` gains `uiMode`/`modeTransition`, `onHudMode`/`setHudInteractive` effects, and HUD wiring of existing state (tasks, transcript, hand control, wake/sleep, mute). PO question banner rendered inside HudShell (Claude-specific delta vs upstream).
- **Main process**: `electron/main.mjs` — overlay window flags, `setIgnoreMouseEvents` handling, `hud:toggle`/`hud:interactive`/`hud:mode` IPC, Tray + `updateTrayMenu()`, `globalShortcut` (register on ready, unregister on will-quit), `iris:wake` emit. `electron/preload.cjs` — `toggleHud`, `setHudInteractive`, `onHudMode`, `onWakeRequest` (+ `windowControl` if adopted).
- **Assets/build**: `build/trayTemplate.png`, `build/trayTemplate@2x.png`, `scripts/render-icon.mjs`; electron-builder `files` list gains the tray images (upstream package.json delta).
- **Config/docs**: `IRIS_HUD_HOTKEY` (default `Alt+Space`) documented in `.env.example`; README gains the HUD mode notes.
- **Dependencies**: none new (`lucide-react` already present).
- **Prerequisites**: all three prior changes implemented (they are — archived 2026-07-17/18).
